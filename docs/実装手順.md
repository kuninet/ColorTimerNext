# 実装手順

## 進め方
* githubリポジトリ(公開)を作成
* 設計文書、実装計画文書を作成して、承認を取る
* 実装計画に基づいて、githubにissueを作成して、順番に実装する
* issueが実装計画と合っているか確認して承認を取る
* issueごとにブランチを作成して実装を開始して、実装完了ごとに承認をとり、承認されたらPRを作成してマージする。
* **[追加提案]** PR作成時はIssue番号を紐付け（`Fixes #Issue番号` 等）、マージ時にIssueが自動クローズされるようにする。
* **[追加提案]** 実装の完了・マージに合わせて、システム構成や設計に変更があった場合は `README.md` や設計ドキュメントを忘れずに更新する。
* **[追加提案]** 必要に応じて、PR作成時に静的解析（Lint等）が走るCI環境（GitHub Actions等）を設定し、品質を担保する。
* **[追加提案]** クラウド側（LambdaやAPI等）の実装においては、デプロイ前にパソコン上（ローカル環境）で単体テストやモックを利用したテストを実行・検証できる設計とし、実装と並行してテストコードを作成する。
