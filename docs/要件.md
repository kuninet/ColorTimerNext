# ColorTimer Next 要件

## 概要
* ColorTimerの次期バージョンを開発する。
* **プロジェクトの前提**
  * 1台の機器からテスト的にAWSにデータを送ることを想定。
  * エッジ側もクラウド側も**限りなくシンプルな実装**にする。
  * AWSの利用料金は**限りなく無料枠に納める**ことを目標にする。
  * プロジェクトは**オープンソースハードウェア/ソフトウェアとしてGitHubで公開・管理**する。シークレット情報の取り扱いには注意する。
  * 【エージェント開発ルール】調査などのタスクメンバ作業は「Gemini Flash」などの軽量モデルで実施する（今後の他プロジェクトも共通）。

## 要件
### 機能要件
* エッジ側
  * Color Timerのエッジ部分はESP32-WROOM-32を使用する。
    * [[116108]ESP32-WROOM-32Eマイコンボード](https://akizukidenshi.com/catalog/g/g116108/)
  * トリガーボタンを押すと、「作業開始」「作業終了」をクラウドへ送信する。
  * 本体にはLED2つ装備
    * 赤LED - 作業してない
    * 青LED - 作業中(設定したタイムアウト時間が来ると点滅してお知らせ。自動終了はせず、ユーザーのボタン操作で終了イベントを送信する)
  * OLEDディスプレイに現在の状況を表示
    * 起動完了
    * 作業開始
    * 作業終了
    * タイムアウト
    * エラー（Wi-Fi未接続やAWS送信エラーが発生した場合）
  * 圧電ブザーで作業開始、タイムアウト、作業終了を通知
  * **ネットワーク接続・初期設定**: 
    * 初期起動時はマイコンがアクセスポイント(AP)として動作する。
    * PCやスマートフォンのブラウザからアクセスし、Wi-Fi設定（SSID/パスワード）や各設定値（タイムアウト値など）を設定可能にする。

* クラウド側
  * データ受信のアーキテクチャ
    * **API Gateway (HTTP API) + Lambda** 構成で受信する。（AWS IoT Core等のような複雑な証明書管理は避け、ESP32からのHTTPリクエストでシンプルに受信する）
  * 受信した「作業開始」「作業終了」の日時を永続化する。
    * クラウド側での受信時刻をタイムスタンプとする。（エッジ側では時刻を持たず、「開始」「終了」のイベント種別のみを送信する）
    * ストレージには **DynamoDB**（オンデマンド または プロビジョニング25WCU/25RCUの無料枠内）を使用する。
  * Webアプリ
    * 作業履歴の一覧表示できるようにしたい。
    * 統計表示（週次、月次の稼働状況）を見えるようにしたい。
    * ホスティングは **S3静的ホスティング + CloudFront** のSPA構成とする。

### 非機能要件
* セキュリティ対策
  * 最低限のセキュリティはかけたい。AWSにいたずらされない程度の対策をしたい。軽く機器認証はしたい。
    * API Gatewayの **APIキー** を利用して、エッジ側からのリクエストを簡易認証する。
  * Webアプリを使えるIDを限定したい。アプリID管理はあまり複雑になりすぎない程度で。
    * Cognito等の複雑な構成は避け、**CloudFront Functionsを使ったBasic認証** で保護する。
    * ※ 運用保守用として、パスワード変更やID/PW追加ができる簡易なシェルスクリプトを別途用意する。
  * **OSS対応**: 認証情報や設定値などは.env等で管理し、リポジトリには含めない。

* パフォーマンス
  * 当初は1台程度の負荷を想定。

* 可用性
  * 個人使用なので、可用性要件なし。

* コスト
  * AWSのサービスを活用するが、なるべく安価にしたい。無料枠にすべて収まるのが理想。
