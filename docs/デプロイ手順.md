# デプロイ・構築手順

本ドキュメントでは、ColorTimer Nextプロジェクトの各コンポーネント（クラウド基盤・Webアプリ・エッジデバイス）のビルドから実環境へのデプロイ（書き込み）手順を記述します。
開発機材として、macOS および Windows 双方での操作を想定しています。

---

## 1. クラウド基盤のデプロイ (AWS SAM もしくは CDK想定)
**構成**: API Gateway + Lambda + DynamoDB

### 前提条件
- [AWS CLI](https://aws.amazon.com/jp/cli/) およびデプロイツール（[AWS SAM CLI](https://docs.aws.amazon.com/serverless-application-model/latest/developerguide/install-sam-cli.html) または Node.js + AWS CDK）がインストールされていること。
- `aws configure` コマンドで対象AWSアカウントのCredentials設定が完了していること。

### デプロイコマンド (SAM CLIの例)
```bash
# プロジェクトのバックエンドディレクトリへ移動
cd backend

# ビルドの実行 (mac/win共通)
sam build

# デプロイ (初回は --guided オプションで設定を保存)
sam deploy --guided
# 2回目以降
sam deploy
```

---

## 2. Webアプリのデプロイ (フロントエンド)
**構成**: S3 (静的ホスティング) + CloudFront

### 前提条件
- Node.js (v18以降推奨) がインストールされていること。
- クラウド基盤側でデプロイされた S3バケット名 と CloudFrontの Distribution ID を確認していること。

### デプロイコマンド
```bash
# プロジェクトのフロントエンドディレクトリへ移動
cd frontend

# 依存モジュールのインストール (mac/win共通)
npm install

# 本番用ビルド
npm run build

# --- AWSへのアップロードとキャッシュクリア ---

# macOS / Linux / Windows(PowerShell/GitBash共通)
# S3バケットへ同期 (ビルド成果物の `dist` や `out` フォルダから)
aws s3 sync ./dist s3://<対象S3バケット名> --delete

# CloudFront のキャッシュ無効化
aws cloudfront create-invalidation --distribution-id <Distribution ID> --paths "/*"
```
*(※必要に応じてこれらの aws コマンドを `npm run deploy` などのスクリプトとして `package.json` にまとめておくことを推奨します。)*

## 3. エッジデバイスの書き込み (ESP32)
**構成**: ESP32-WROOM-32 (PlatformIO での開発を想定)

### 前提条件
- VSCode および [PlatformIO IDE 拡張機能](https://platformio.org/install/ide?install=vscode) がインストールされていること。（CLI版の `pio` コマンドでも可）
- ESP32とPCがUSBケーブル(データ通信対応)で接続されていること。
- **Windows特有**: ESP32のUSBシリアル変換ドライバ (CP210x等) がインストールされ、デバイスマネージャーでCOMポートとして認識されていること。

### デプロイコマンド (PlatformIO Core CLI の例)
VSCode上からPlatformIOの「Upload」ボタン（右下の矢印アイコン）を押すのが最も簡単ですが、CLIで行う場合は以下となります。

```bash
# プロジェクトのエッジ側ディレクトリへ移動
cd edge

# 依存ライブラリのインストール・ビルド
pio run

# ESP32実機への書き込み
pio run -target upload

# (オプション) シリアルモニタを開いて動作確認・ログ確認
pio device monitor
```
- ※Windowsでは対象ポートが複数ある場合、上記コマンドで自動で認識されないことがあります。その場合は `pio run -t upload --upload-port COM3` のように明示的に指定してください。
- ※macOSでは `--upload-port /dev/cu.usbserial-*` のように指定できます。
