# 実装計画 (Issue分割予定一覧)

システムの構築を「クラウド基盤」「エッジ」「フロントエンド」の3つのフェーズに分け、以下の粒度でGitHub Issueを作成し、順次実装とPR（Pull Request）を進める計画とします。

## Phase 1: クラウド基盤 (AWS バックエンド)
このフェーズでは、AWS上にAPIとデータベースを構築し、ローカルやPostman等で単体テストが通る状態を目指します。

* **Issue 1**: [Infra] AWSバックエンドリソースのコード化（SAM等）および DynamoDBテーブルの作成
* **Issue 2**: [Backend] ステータスデータ受信用機能 (API Gateway + Lambda) の実装とローカル単体テスト・モックテストの作成
* **Issue 3**: [Backend] データ取得用API (API Gateway + Lambda) の実装と単体テスト・CORS設定

## Phase 2: エッジデバイス (ESP32)
APIのインターフェースが固まった状態で、実際にデバイスからの通信とハードウェア制御を組み込みます。

* **Issue 4**: [Edge] PlatformIOプロジェクトの初期化と、`WiFiManager` を用いたAPモード・ブラウザ設定(Wi-Fi・APIキー・タイムアウト等)の実装
* **Issue 5**: [Edge] ハードウェア制御 (物理ボタンによる状態切替、フルカラーLED制御、ブザー鳴動、OLEDディスプレイへの状態表示) の実装
* **Issue 6**: [Edge] AWS APIへのHTTP POST送信処理と、エラー時（未接続・送信失敗）のOLEDエラー表示・ハンドリングの実装

## Phase 3: フロントエンド (Webアプリケーション)
クラウドのAPIからデータを取り出し、描画と集計を行うSPAを構築します。

* **Issue 7**: [Frontend] React(Vite等)プロジェクトの初期化と、S3+CloudFrontへの静的デプロイパイプライン（手動コマンド等）の準備
* **Issue 8**: [Frontend/Infra] CloudFront Functions を用いたサイト全体のBasic認証の実装と、簡易なID/PW管理用シェルスクリプトの作成
* **Issue 9**: [Frontend] 作業履歴・ログの一覧表示画面（API連携）の実装
* **Issue 10**: [Frontend] 作業ログに基づく週次・月次の稼働状況・統計表示画面の実装

---
本計画に基づいて順番にIssueを作成し、実装・レビュー（ユーザー確認）・マージのサイクルを回して開発を進めます。
